# 二期管理平台规划

二期目标是在一期 Docker Compose 稳定运行的基础上，增加一个轻量 Web 管理平台，用于查看、编辑和控制 10 个 `frpc` 实例。

二期必须继续以 Docker Compose 作为唯一执行层，不再引入 systemd 作为等价后端，避免长期维护两套控制逻辑。

## 目标能力

- 查看 10 个实例的运行状态。
- 查看每个实例的 CPU、内存、重启次数和运行时间。
- 启动、停止、重启单个实例。
- 查看单个实例最近日志。
- 生成和编辑 `frpc.toml`。
- 配置变更前自动备份。
- 从备份恢复配置。
- 支持 Telegram、邮箱或 Webhook 告警。
- 记录关键操作审计日志。

## 建议架构

- 后端：运行在 VPS 本机的小型服务，工作目录固定为 `/opt/frpc-multi`。
- 前端：带登录认证的 Web 面板，只监听 localhost、内网地址或 VPN 地址。
- 配置源：`configs/*/frpc.toml` 继续作为真实配置源。
- 状态源：通过 Docker Compose / Docker API 读取容器状态。
- 存储：可选 SQLite，用于保存审计日志、告警配置和界面元数据。
- 执行方式：后端只允许执行受控的 Compose 操作，不提供任意 shell 命令入口。

## 推荐页面

- 总览页：显示 10 个实例状态、资源占用、重启次数和最近异常。
- 实例详情页：显示配置摘要、最近日志和操作按钮。
- 配置编辑页：编辑 TOML，并在保存前做基础校验。
- 备份页面：查看、下载、恢复历史配置备份。
- 告警设置页：配置通知渠道和触发条件。
- 审计日志页：记录谁在什么时候执行了什么操作。

## 安全要求

- 管理平台必须启用登录认证。
- 默认不允许公网直接访问。
- 高风险操作需要二次确认，例如停止实例、恢复备份、批量重启。
- 不在页面明文展示完整 token。
- 后端不能开放任意命令执行能力。
- 所有配置变更必须先备份再生效。

## 非目标

- 二期第一版不做多 VPS 集群管理。
- 不把 systemd 作为第二套正式执行后端。
- 不做公开免登录状态页。
- 不把 frps 服务端管理混入第一版。
- 不做复杂的租户权限系统，除非后续明确需要多人协作。

## 一期到二期的衔接

一期已经确定以下稳定边界：

- 运行目录为 `/opt/frpc-multi`。
- `compose.yaml` 管理全部实例。
- `configs/*/frpc.toml` 是配置源。
- `scripts/backup-configs.sh` 负责备份。
- `scripts/check-health.sh` 负责基础巡检。

二期管理平台应复用这些边界，而不是重新设计部署方式。
